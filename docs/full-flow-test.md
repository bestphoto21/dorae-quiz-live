# 전체 행사 플로우 테스트

## 테스트 목적

Dorae Quiz Live의 참가자 등록, 퀴즈 진행, Q&A, 럭키드로우, 운영 로그, 리허설, 헬스체크가 현장 운영 흐름에서 안전하게 동작하는지 확인한다. 기능 확인과 함께 개인정보 미노출, 정답 공개 전 정답 미노출, 버튼/배지/안내 박스의 가독성을 함께 점검한다.

## 테스트 전 준비

- 테스트용 행사 코드, 관리자 계정, 참가자 테스트 기기 1대 이상, 스크린 확인용 브라우저를 준비한다.
- `.env.local` 값이나 운영 환경변수 값은 화면 캡처, 채팅, 문서에 붙여 넣지 않는다.
- 테스트 중 참가자 연락처, 이메일, 인증 토큰, 원본 송출 보조 데이터가 공개 화면과 로그 화면에 보이지 않는지 확인한다.
- 관리자 화면, 참가자 모바일 화면, 스크린 화면을 동시에 열어 상태 전환을 확인한다.

## 행사 생성 확인

1. `/admin/events`에서 새 행사를 만든다.
2. 행사 목록에 활성 상태, 행사 코드, 장소, 시작/종료 시간이 읽기 쉬운 대비로 표시되는지 확인한다.
3. 행사 상세 화면에서 참가자 URL, 등록 URL, 참여 URL, 스크린 URL을 확인한다.

## 퀴즈 세션/문제 확인

1. 행사 상세 화면에서 문제 관리로 이동한다.
2. 퀴즈 세션을 만들고 상태를 준비 상태로 바꾼다.
3. 문제와 선택지 4개, 제한 시간, 정답을 입력한다.
4. 관리자 화면에서 정답은 보이지만 참가자/스크린 공개 전 응답에는 정답 값이 포함되지 않는지 확인한다.

## 참가자 등록 확인

1. `/e/[eventCode]/join`에 모바일 폭으로 접속한다.
2. 이름과 휴대폰 번호를 입력하고 개인정보 동의 후 등록한다.
3. 등록 버튼, 입력창 placeholder, 안내 문구가 모바일에서 읽기 쉬운지 확인한다.
4. 공개 화면에는 휴대폰 번호나 내부 정규화 값이 보이지 않는지 확인한다.

## 참가자 play 접속 확인

1. `/e/[eventCode]/play`에 접속한다.
2. 대기 상태에서 자동 새로고침 안내와 Q&A 질문 입력 영역이 보이는지 확인한다.
3. 버튼 높이와 터치 영역이 모바일에서 충분한지 확인한다.

## 대기 화면 송출

1. 라이브 콘솔에서 대기 화면 송출을 누른다.
2. `/screen/[eventCode]`에서 대기 화면, 행사명, 참가자 입장 경로가 크게 보이는지 확인한다.

## 휴식 화면 송출

1. 라이브 콘솔에서 휴식 화면 송출을 누른다.
2. 스크린에 휴식 시간 문구가 한국어로 표시되고 내부 상태값이 노출되지 않는지 확인한다.

## 퀴즈 문제 송출

1. 라이브 콘솔에서 문제를 선택해 시작한다.
2. 참가자 화면에 문제와 선택지가 표시되는지 확인한다.
3. 스크린 화면에 문제, 선택지, 남은 시간이 16:9 화면에서 읽기 쉬운지 확인한다.

## 참가자 답변 제출

1. 참가자 화면에서 선택지를 누른다.
2. 제출 중, 응답 완료, 정답 공개 전 안내 문구가 자연스럽게 표시되는지 확인한다.
3. 스크린의 총 응답과 선택지별 응답 수가 갱신되는지 확인한다.

## 중복 답변 방지 확인

1. 같은 참가자로 같은 문제에 다시 답변을 시도한다.
2. 중복 답변이 저장되지 않고 참가자에게 이해 가능한 안내가 표시되는지 확인한다.

## 마감 후 답변 방지 확인

1. 라이브 콘솔에서 응답 마감을 누른다.
2. 참가자 화면에서 선택 버튼이 비활성화되고 버튼 의미가 읽히는지 확인한다.
3. 마감 후 새 답변이 저장되지 않는지 확인한다.

## 정답 공개 전 정답 미노출 확인

1. 정답 공개 전 참가자 상태 API와 스크린 상태 API 응답을 확인한다.
2. 정답 필드가 포함되지 않는지 확인한다.
3. 스크린 화면에서도 정답 강조가 표시되지 않는지 확인한다.

## 정답 공개 후 정답 표시 확인

1. 라이브 콘솔에서 정답 공개를 누른다.
2. 참가자 화면에 정답과 내 선택 결과가 표시되는지 확인한다.
3. 스크린 화면에 정답과 정답자 수가 표시되는지 확인한다.

## Q&A 질문 제출

1. 참가자 play 화면에서 질문을 입력하고 제출한다.
2. 제출 완료와 검토 중 상태 문구가 자연스러운지 확인한다.
3. 질문 제출 버튼이 모바일에서 전체 폭 또는 충분한 터치 영역을 갖는지 확인한다.

## 미승인 질문 스크린 미노출 확인

1. 관리자 Q&A 화면에서 새 질문이 검토 중으로 표시되는지 확인한다.
2. 승인 전에는 스크린에 질문 원문이 표시되지 않는지 확인한다.

## 승인 질문 스크린 송출 확인

1. 관리자 Q&A 화면에서 질문을 승인한다.
2. 승인 질문 스크린 송출을 누른다.
3. 스크린에는 승인된 질문, 표시 이름, 소속/팀만 표시되는지 확인한다.

## 럭키드로우 준비 화면 확인

1. 라이브 콘솔 또는 추첨 화면에서 럭키드로우 준비 화면을 송출한다.
2. 스크린에 준비 화면이 표시되고 개인정보가 보이지 않는지 확인한다.

## 당첨자 송출 확인

1. 경품을 등록하고 추첨을 실행한다.
2. 당첨 결과가 저장된 뒤 스크린에 당첨자 표시 이름과 경품명만 표시되는지 확인한다.
3. 수령 완료, 당첨 취소, 재추첨 버튼의 문구와 대비가 명확한지 확인한다.

## 운영 로그 확인

1. 운영 로그 화면에서 최신 로그를 확인한다.
2. 질문 원문, 연락처, 이메일, 인증 정보, 원본 송출 보조 데이터가 표시되지 않는지 확인한다.
3. action과 detail은 운영자가 이해 가능한 문구로 표시되는지 확인한다.

## 리허설 체크 확인

1. 리허설 체크 화면을 연다.
2. 행사 기본 상태, 참가자 등록, 퀴즈 준비, Q&A, 럭키드로우, 스크린 송출 상태가 정상/주의/위험으로 구분되는지 확인한다.
3. 바로가기 버튼 라벨과 대비가 명확한지 확인한다.

## 헬스체크 확인

1. `/admin/health`를 연다.
2. 환경변수는 등록 여부만 표시되고 실제 값은 표시되지 않는지 확인한다.
3. Supabase 연결과 events 테이블 조회 상태가 값 노출 없이 표시되는지 확인한다.

## 개인정보 미노출 확인

1. 참가자 공개 화면, 스크린 화면, screen API, participant API, 운영 로그, 헬스체크 화면을 확인한다.
2. 참가자 연락처, 이메일, 인증 토큰, 환경변수 값, 원본 송출 보조 데이터가 표시되지 않는지 확인한다.
3. Q&A 질문은 승인 전 스크린에 표시되지 않는지 확인한다.

## 텍스트 대비/버튼 가독성 확인

1. primary, secondary, outline, ghost, 링크형 버튼을 기본/hover/focus/disabled 상태에서 확인한다.
2. 상태 배지, 안내 박스, 카드 내부 라벨, 입력창 placeholder가 배경 위에서 명확히 보이는지 확인한다.
3. 참가자 모바일 CTA와 관리자 우측 액션 버튼 라벨이 흐리거나 배경과 섞이지 않는지 확인한다.
4. 스크린 화면의 상단 상태 배지와 통계 라벨이 16:9 화면에서 멀리서도 읽히는지 확인한다.

## 실패 시 확인 순서

1. 행사 코드와 활성 상태를 확인한다.
2. 참가자 등록 여부와 세션 쿠키 상태를 확인한다.
3. 라이브 콘솔의 현재 운영 모드와 현재 송출 화면을 확인한다.
4. 현재 세션과 현재 문제가 선택되어 있는지 확인한다.
5. 정답 공개 상태와 결과 표시 상태를 확인한다.
6. Q&A 질문이 승인 상태인지 확인한다.
7. 럭키드로우 경품 수량과 기존 당첨 이력을 확인한다.
8. 운영 로그에서 민감정보 없이 허용된 요약 키만 표시되는지 확인한다.
9. 헬스체크에서 필수 환경 등록 여부와 Supabase 연결 상태를 확인한다.
